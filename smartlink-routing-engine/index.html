01 / Operational context

Eligibility prevents impossible routes

Country, device, traffic restrictions, offer availability and cap status are hard constraints. Performance history cannot override them.

02 / Operational context

Segmentation creates comparable traffic groups

GEO, device and traffic type influence user behaviour and funnel compatibility. Source and placement subIDs make reporting more useful without exposing personal user data.

03 / Operational context

Fallback protects coverage

When the preferred destination is paused or ineligible, the click should move to the next compatible route rather than fail or land on an unsupported page.
